# 周报格式参考

这个参考文件固化周报的目标格式。它定义输出结构，不负责提供周报数据。

默认数据源：

- 周报优先基于指定仓库的一周提交记录生成
- 可辅以工作区未提交改动判断未收尾事项
- 不默认依赖日报、项目总结或其他二次整理文档

## 核心要求

整份周报是一个统一叙事，不按仓库拆顶层。结构由四个固定章节构成，固定顺序：

1. `本周进展概述`
2. `本周进展明细`
3. `下周里程碑`
4. `下周计划明细`

每一章节使用混合层级列表：

- 顶层：有序列表 `1.` / `2.` / `3.` / `4.`
- 工作流层（在 `本周进展明细` 与 `下周计划明细` 下）：字母列表 `a.` / `b.` / `c.` / `d.`
- 明细层（在 `本周进展明细` 的工作流下）：罗马数字列表 `ⅰ.` / `ⅱ.` / `ⅲ.` / `ⅳ.`
- 缩进：每层 2 个空格

## 列表结构

```markdown
1. 本周进展概述：[一句话概述]
2. 本周进展明细：
  a. [工作流 / 主题 1]
    ⅰ. [细节 1]
    ⅱ. [细节 2]
  b. [工作流 / 主题 2]
    ⅰ. [细节 1]
  c. [工作流 / 主题 3]
    ⅰ. [细节 1]
  d. 其他
    ⅰ. [杂项 / 打包 / 运维]
3. 下周里程碑：[一句话概述]
4. 下周计划明细：
  a. [具体工作项]
  b. [具体工作项]
```

- 不要把正文写成纯段落
- 不要把正文写成 `##` 二级标题分节
- 不要在顶层按仓库拆分；跨仓共享主题归到同一工作流
- 不要把 `下周里程碑` 和 `下周计划明细` 聚合到正文最后
- 四个固定章节的数量与顺序不可改变
- 必要的杂项、打包修复、运维问题统一收到 `其他` 工作流

### 1. 本周进展概述

- 一句话概述
- 重点回答：
  - 什么事情做到了什么程度
  - 上了什么内容或能力
  - 当前使用、试用、验证或推广状态如何

### 2. 本周进展明细

- 按工作流 / 主题分组，3 到 5 个 `a.` 级条目
- 每个工作流下放 2 到 5 个 `ⅰ.` 级明细
- 优先按主题归组，不按时间顺序平铺
- 跨仓库的同主题工作合并到同一工作流（例如 `协议 Mock 调试` 可包含后端与前端改动）

### 3. 下周里程碑

- 一句话概述
- 重点回答：
  - 下周最值得期待的结果是什么
  - 哪个关键节点最值得写出来

### 4. 下周计划明细

- 用 `a.` / `b.` / `c.` 列具体工作项
- 保持可执行、可验证

## 头部信息

```markdown
# 陈洁周报

- 周期：YYYY-MM-DD 至 YYYY-MM-DD
- 范围：[本周覆盖的工作流 / 模块主题]
- 统计口径：本周本人 Git 提交、相关调研 / 设计文档、本地工作区变更检查
- 来源：Git 提交记录、[关键文档路径或模块]
- 使用技能：`weekly-report-template`
```

- `范围` 写工作流 / 主题，不是裸仓库列表
- `统计口径` 写清评估周报内容的证据来源
- `来源` 写实际拉取的输入（git log + 关键文档目录）
- `使用技能` 固定为 `` `weekly-report-template` ``

## 改写要求

- 原始日报或项目总结不符合这个结构时，必须重组后再输出
- 原始输入如果是 Git 提交记录，必须先按主题归组，再输出
- 原始材料写得很散时，先按主题合并，再写概述和明细
- 没有证据支持的"使用情况""上线效果"不能编造
- 没有明确下周计划时，可以从未完成事项中提炼计划，但要保持为计划表述
- 同一周内的多个主题，按工作流并列在 `2.` 下，不要回退到按仓库拆顶层
- 用户指定了仓库范围时，只能写这些仓库的内容

## 反模式（必须避免）

正文严禁出现仓库底层细节：

- 不出现 commit hash（如 `d8d6ccf`）、commit message 引号原文（如 `feat(mock): 支持...`）、shortlog 摘录
- 不出现文件数 / 代码行数 / `+1240 / -218` 这类 diff 统计
- 不出现裸文件路径（如 `apps/server/app/lib/agent/agent-gateway.ts`）；要写路径时融入到说明里，例如「新增构建监控模块，承担打包过程的 iTrace 上报」
- 不写时间流水账（不要按"周一 → 周二 → 周三"展开）
- 不一条 commit 一个 bullet；先按主题归组再写

正确做法是把多条相关 commit 合成一句话工作流描述，明细层只写「做了什么 + 起到什么作用」。

## 适用说明

- 这个格式默认用于 `陈洁` 的周报
- 外部语雀模板可以作为辅助参考
- 如果外部模板页面无法访问，仍然以本文件的格式要求为准
