# Default Style

Use this style only when the prompt explicitly asks for `default` or stronger semantic phase colors.

## Tokens

- Page background: `#f8fafc`
- Surface background: `#ffffff`
- Surface border: `#cbd5e1`
- Grid stroke: `#d8e0ea`
- Primary text: `#0f172a`
- Secondary text: `#475569`
- Subtle text: `#64748b`
- Neutral connector: `#475569`

## Semantic Palette

- Compile / Infra / Amber: fill `rgba(251,191,36,0.18)`, stroke `#d97706`
- Runtime / Process / Emerald: fill `rgba(52,211,153,0.14)`, stroke `#059669`
- Render / Transport / Cyan: fill `rgba(34,211,238,0.14)`, stroke `#0891b2`
- Reactive / State / Violet: fill `rgba(167,139,250,0.14)`, stroke `#7c3aed`
- Commit / Alerts / Rose: fill `rgba(251,113,133,0.14)`, stroke `#e11d48`
- External / Generic / Slate: fill `rgba(148,163,184,0.16)`, stroke `#64748b`

## Geometry

- Corner radius: `6` to `10`
- Primary stroke width: `1.5` to `1.8`
- Region stroke width: `1.6`
- Label chip radius: `4`

## Tone

- Semantic, technical, phase-oriented
- Stronger region separation
- Good when the diagram needs clearer category coding at first glance

## Best Fit

- architecture
- flowchart
- data-flow
- framework principle diagrams
