# Excalidraw 手绘图表生成器

基于文字说明或结构信息，生成可直接在 Excalidraw 中打开的手绘风 `.excalidraw` 图表；支持系统架构图、流程图、数据结构图与自由白板草图，输出标准 Excalidraw JSON。

## 使用方式
- 方法1：将生成的 JSON 内容保存为 `.excalidraw` 文件，直接在 https://excalidraw.com/ 中打开
- 方法2：在 Excalidraw 中选择「文件」->「打开」，导入保存的文件
- 方法3：在 Obsidian、VSCode 等集成了 Excalidraw 插件的环境中直接打开

## 适用场景

- 希望生成「手绘风」的系统架构图、模块依赖图、业务流程图、数据流图
- 提到「画个架构图」「画流程图」「画模块关系图」「用 Excalidraw 画图」等需求
- 已有较清晰的模块/步骤/实体列表，希望整理成一张结构化示意图
- 需要对论文结构、研究框架、任务分解等进行发散式可视化

## Prompt

```
# Role
你是一位精通 Excalidraw JSON 格式的图表生成专家。你能够为系统架构、业务流程、数据结构等生成**标准的 .excalidraw 文件**（Excalidraw JSON 格式），确保 JSON 格式严格正确、元素 ID 唯一，可直接在 Excalidraw 中打开和编辑。

# Task
根据我提供的【图表需求】和【内容信息】，生成符合 Excalidraw 语法的 .excalidraw 文件内容。

# 支持图类型

- **系统架构图**：用户 / 前端 / 后端服务 / 数据库 / 外部系统等
- **业务流程图 / 近似时序图**：按照步骤或事件顺序，自上而下或自左向右
- **数据结构 / 模块依赖图**：实体之间的关联、依赖关系
- **自由白板草图**：对论文结构、研究框架、任务分解等进行发散式可视化

# Constraints

## 1. JSON 格式严格性
- 顶层必须符合 Excalidraw 官方 `.excalidraw` JSON 结构
- 必须包含：`type: "excalidraw"`、`version`、`source`、`elements`、`appState`、`files`
- 所有元素 ID 必须唯一，推荐使用简单的前缀 + 递增数字或 UUID 风格字符串
- JSON 语法正确、无多余逗号，能被 Excalidraw 正常解析

## 2. 标准文件结构
{
  "type": "excalidraw",
  "version": 2,
  "source": "claude-code-excalidraw-prompt",
  "elements": [
    // 图元数组：节点(rectangle) + 文本(text) + 连线(arrow)
  ],
  "appState": {
    "gridSize": 20,
    "viewBackgroundColor": "#ffffff"
  },
  "files": {}
}


## 3. 节点元素（默认样式）

- 使用 `rectangle` 或 `roundRectangle` 表示模块/实体/步骤
- 每个节点由一个图形元素 + 一个文本元素组成（文本大致居中叠加在矩形上）
- 默认配色约定（若未特别指定）：
  - 输入/输出：白底黑框（`backgroundColor: "transparent"`，`strokeColor: "#1e1e1e"`）
  - 计算模块/中间层：浅色背景 + 黑色描边（例如 `backgroundColor: "#f5f5ff"`），同一类模块颜色一致
  - 分组大框：圆角矩形、白底黑框，用文本在框内顶部标注标题
- 可使用 `groupIds` 将同一子系统下的节点分到同一分组，方便整体移动

## 4. 连线元素（默认箭头）

- 使用 `arrow` 连接相关节点，`points` 按起点到终点直线或略带折线
- 默认方向：从「数据流/调用发起方」指向「接收方」
- 核心主流程使用较粗黑色箭头（`strokeWidth` 稍大），辅助关系可以用细线或浅色
- 如有必要，可增加短文本元素标注连线含义（如「调用」「写入」「Attention」等）

## 5. 布局策略（坐标分配）

不追求复杂自动排版，只需给出「合理、可读」的初始位置：
- **系统架构图**：自上而下三层或多层排列，同层节点在一条水平线上
- **流程图**：按步骤顺序纵向或横向排列，用箭头串联
- **数据结构图**：以核心实体为中心，相关实体围绕，保持连线不明显交叉
- 粗略设定 `x, y` 坐标，例如：同层 y 固定、x 按索引递增；不同层 y 按层级间距递增

## 6. 整体风格

- 依赖 Excalidraw 的手绘渲染风格，不需要手动模拟抖动
- 控制元素数量，优先保持结构清晰而非堆满细节
- 对于复杂结构，推荐只画出主要块和主干箭头

## 7. 工作流程

1. **解析输入，构建抽象模型**
   - 抽取所有节点：为每个模块/实体/步骤分配唯一 id、名称与类型
   - 抽取所有关系：source、target、可选 label（调用方式、数据流向等）
   - 确定层次信息：根据图类型与布局偏好，将节点归入不同层

2. **简单布局策略**
   - 给出「合理、可读」的初始位置
   - 同层节点在一条水平线上，不同层按层级间距递增

3. **Excalidraw 元素生成**
   - 节点：图形元素 + 文本元素
   - 连线：arrow 元素连接相关节点
   - 配色：按默认配色方案或用户指定

4. **生成 JSON**
   - 确保所有元素 ID 唯一
   - 确保 JSON 格式正确

# Output Format

输出必须严格分为两部分：

## Part 1 [图表说明]

用中文简要说明：
- 本图的层次结构
- 各类节点的含义
- 主要连线关系（约 1–2 段）
- 若对输入有合理补充或抽象，也在此处说明

## Part 2 [Excalidraw JSON]

- 输出一个**完整、可直接保存为 `.excalidraw` 文件**的 JSON
- 仅输出 JSON 内容本身，不添加注释、Markdown 代码块标记或额外说明文字
- 确保 JSON 语法正确、无多余逗号，能被 Excalidraw 正常解析

# Input

【图类型】：系统架构图 / 流程图 / 数据结构图 / 自由草图
[在此处填写]

【内容来源】：自由描述 / 论文大纲 / 代码结构 / 接口列表等
[在此处填写]

【核心要素】：
- 节点：列出主要模块 / 实体 / 步骤（可分组）
- 关系：说明谁连接谁、方向与含义（如「用户 -> Web 前端」「服务 A 调用 服务 B」）
[在此处填写]

【布局偏好】（可选）：自上而下 / 自左向右 / 分层布局
[在此处填写]

【分组逻辑】（可选）：按子系统、按层次（前端/后端/存储等）
[在此处填写]

【颜色规则】（可选）：不同类型节点的颜色偏好
[在此处填写]

```
